react-native-touch-id使用

发布 : 2019-04-01 浏览 :

react-native-touch-id 使用

一、安装

yarn add react-native-touch-id

二、链接到工程

react-native link react-native-touch-id

三、配置指纹传感器权限

1）Android

在 Project 浏览模式下，进入 app 下的 src 下的 main 文件夹，打开 AndroidManifest.xml 文件，在文件根节点下加入：

<uses-permission android:name="android.permission.USE_FINGERPRINT" />

2）iOS

打开 iOS 工程中的 info.plist 文件，在文件根节点下加入：

Privacy - Face ID Usage Description

四、具体使用

简单示例

import React from "react";
import { Alert } from "react-native";
import TouchID from "react-native-touch-id";

class YourComponent extends React.Component {
    _pressHandler() {
        const optionalConfigObject = {...};
        TouchID.authenticate('auth demo', optionalConfigObject)
        .then(success => {
            Alert.alert('Authenticated Successfully');
        })
        .catch(error => {
            Alert.alert('Authentication Failed');
        });
    },

    render() {
        return (
            <View onPress={() => {this._pressHandler()}}>
                <Text>
                    Authenticate with Touch ID
                </Text>
            </View>
        );
    }
}

方法

  • authenticate(reason, config)
    方法描述：尝试使用Face ID/Touch ID进行身份验证。返回一个Promise对象。
  • isSupported(config)
    方法描述：返回一个 Promise 对象，如果不支持指纹识别，则不返回Promise 对象，在 iOS 平台，biometryType 返回为 String 类型的 FaceID 或者 TouchID。

参数

  • reason - String - 为请求身份验证提供一个清晰的缘由（可选）
  • config - Object - 为对话框配置更详细的参数（可选）
    • title - String - 对话框确认标题（仅安卓适用）
    • imageColor - String - 指纹图像颜色（仅安卓适用）
    • imageErrorColor - String - 尝试失败后指纹图像颜色（仅安卓适用）
    • sensorDescription - Strng - 显示在指纹图像旁边的文字（仅安卓适用）
    • sensorErrorDescription - String - 尝试失败后显示在指纹图像旁边的文本（仅安卓适用）
    • cancelText - String - 取消按钮文本（仅安卓适用）
    • fallbackLabel - String - 默认情况下指定“显示密码”标签，如果设置为空字符串标签是不可见的（仅 iOS 适用）
    • unifiedErrors - Boolean - 是否返回统一错误信息（默认为 false）
    • passcodeFallback - Boolean - 是否允许适用键盘密码（默认为 false）
